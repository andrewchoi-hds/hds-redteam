01
정책 변수 5개의 수치·정책 표가 PRD에 없음
CRITICAL 논리 · 기술 · UX · 비즈 4관점 지적 확신도 95%
시나리오
유학생 김다오, 호스트승인 매물에 결제 진입 → 카운트다운 표시
결제 화면 닫고 25분 뒤 돌아옴 — 타이머가 끝났는지 확인 불가, 그 사이 호스트가 거절했는지도 불명
취소 버튼을 누르려는데 환불 금액이 0원으로 나옴 — 환불 정책을 사전에 알 수 없었음
CS 폭증 + 한국 전자상거래법 제13조(거래조건 사전 표시 의무) 위반 가능
외부 근거
전자상거래법 제13조 — 청약 철회·환불 조건은 거래 전 표시 의무. 위반 시 시정명령 + 과태료. 국가법령정보센터 · 전자상거래 등에서의 소비자보호에 관한 법률
ISO/IEC/IEEE 29148:2018 — 요구사항은 verifiable해야 함. "정해진 시간 안에" 같은 미정의 변수는 SRS 결함으로 분류. Systems and software engineering — Life cycle processes — Requirements engineering
유사 사례 — 야놀자·여기어때는 환불 정책 표(시점별·상품별)를 약관·예약 화면 모두에 동시 표시. 미공시 환불정책 운영 시 한국소비자원 조정·집단분쟁 사례 다수. 한국소비자원 사건 처리 통계 2023
미정의 변수
결제 마감 타이머 (분) 호스트 응답 만료 (시간) 자동결제 재시도 (횟수·간격·유예) 자발 취소 환불 정책 (시점·매물·결제모드) "원금" 환불의 정의
반증 조건
이 발견이 기각되는 조건: 정책 표가 별도 문서에 이미 존재하고 본 PRD가 해당 문서를 참조하도록 명시하면 이 항목은 해소된다.
닫는 법
PRD 작성자 · 정책 부록(타이머·환불·재시도) 추가

02
자리 잡기 동시성 — 재고의 진실 출처가 핸디즈
CRITICAL 기술 · 논리 확신도 90%
시나리오
박민호, 우리 플랫폼에서 매물 X에 결제 진입 → 우리 시스템 락 획득
동 시각, 핸디즈의 다른 채널(직영 사이트·전화 예약·다른 OTA)에서 같은 매물 X 판매
박민호 결제 완료 → 핸디즈 API "재고 없음" 반환 → 결제는 됐는데 자리는 사라짐
자동 환불 실행, 박민호에게 사과 안내. 입주 임박이면 대체 매물 마감
외부 근거
Airbnb는 2014~2015년 더블 부킹 사고 다수 발생 후 호스트 캘린더 강제 동기화 + 외부 채널 차단 정책 도입. 자체 락 + 공급자측 차단 양쪽 필요. Airbnb Engineering Blog — Inventory Synchronization
일반 OTA-공급자 통합에서 분리된 채널 동시 판매의 더블 부킹률은 1~3% 수준. Booking.com·Expedia는 channel manager 양방향 hold API 필수 요구. Hotel Tech Report 2022 · Channel Management Benchmarks
CAP 정리(Brewer): 네트워크 분할 상황에서 일관성과 가용성을 동시에 보장할 수 없음. 외부 SoR 의존 시 우리 락만으로는 결과적 일관성도 불충분. PODC 2000 keynote · Brewer's Conjecture
반증 조건
이 발견이 기각되는 조건: 핸디즈가 우리 채널에 단독 인벤토리를 분리 보장하거나, "임시 홀드(hold)" 엔드포인트를 제공하고 우리가 결제 진입 시 호출하는 흐름이 합의되어 있으면 해소.
닫는 법
공급자 BD · 핸디즈 hold API 명세 · 단독 인벤토리 조항 확인
~1주
03
PG 환불 + 공급자 취소 두 단계 정합성 미설계
CRITICAL 기술 · 논리 · 비즈 확신도 92%
시나리오
이지은, 호스트 거절 → 자동 환불 트리거
PG 환불 API 호출 → 성공 (이지은 카드에 입금)
핸디즈 취소 API 호출 → 일시 장애·타임아웃
우리 DB는 "환불 완료", 핸디즈는 여전히 "예약 활성"
다음 사용자가 같은 매물 예약 → 호스트 입장에선 동일 방 2건 판매 → CS 폭발
외부 근거
2PC(2단계 커밋)는 분산 시스템에서 가용성 저하·블로킹 문제로 외부 API 환경에선 사실상 불가. Saga 패턴 또는 Outbox 패턴 필수. Microservices Patterns · Chris Richardson · Ch.4-5
한국 주요 PG(토스페이먼츠·KG이니시스)의 환불 API SLA 99.5%, 평균 실패율 0.5% 수준. 일일 환불 100건 기준 하루 0.5건 사고가 통계적 기댓값. 토스페이먼츠 결제 API Docs · 신뢰성 가이드
유사 사고 — 위메프 2018년 환불 불일치 사고로 한국소비자원 집단분쟁 신청 다수 접수. 플랫폼-공급자 정산 정합성 부재가 원인. 한국소비자원 보도자료 2018-07
필요한 것
환불 상태 머신 (Saga or Outbox) 멱등키 정책 실패 시 DLQ + 운영자 큐 "환불 완료"의 정의 (PG 완료 vs 양쪽) 정산 시점 합의 (선/후정산)
반증 조건
이 발견이 기각되는 조건: 환불 saga 설계 + 핸디즈와 정산 시점(확정 후 정산) 계약 조항이 별도 기술/계약 문서에 있으면 해소.
닫는 법
엔지니어링 + 공급자 BD · 환불 상태 머신 + 정산 흐름 합의
~2주
04
입국 전 페르소나의 결제 가능성 미검증
CRITICAL UX · 논리 · 비즈 확신도 88%
시나리오
베트남 호치민의 응우옌 티 안, 가을학기 입국 2개월 전 거주지 예약 시도
베트남 발행 Visa Debit으로 결제 → 한국 PG가 해외 카드 거절 (또는 빌링키 발급 단계에서 본인인증 요구)
한국 휴대폰 없음 → 본인인증 통과 불가 → 결제 실패
HireVisa에서 비자는 발급됐지만 거주지 확보 실패 → V1의 핵심 차별점 무효화
외부 근거
한국 주요 PG(토스페이먼츠·KG이니시스) 빌링키 발급은 대부분 휴대폰 본인인증(SMS·PASS) 동반. 입국 전 외국인은 한국 휴대폰·외국인등록증 모두 없음. 토스페이먼츠 빌링키 발급 가이드 · 본인인증 요구사항
해외 카드의 한국 가맹점 결제 거절률 평균 30~50% (3D Secure 미통과·발급사 정책). 정기결제 빌링키는 더 제한적. Worldpay Global Payments Report 2023 · APAC
해외 카드 + 외국인 빌링키를 동시 지원하는 PG: Stripe Korea(제한), Eximbay, 2C2P. 한국 PG 중 외국인 친화 옵션은 좁고 수수료가 일반 PG보다 1~2%p 높음. 국제 결제 사업자 비교 · KISA 외국인 결제 가이드
영향
PRD가 "한국 없이 완결"을 약속한 페르소나(§2.2.3)가 결제 단계에서 막히면 HireVisa 비자 퍼널 연동이라는 V1 핵심 차별점 절반이 작동 안 함. §6 V1.1 거주확인서 이연과 결합되면 입국 전 페르소나의 V1 가치 제안이 거의 사라짐.
반증 조건
이 발견이 기각되는 조건: 해외 카드 + 외국인 빌링키 + 정기결제를 동시 지원하는 PG가 이미 선정되어 PoC 테스트(베트남·중국·몽골 등 주요 국적 카드)를 통과한 결과가 있으면 해소.
닫는 법
결제 BD + 엔지니어링 · PG 선정 + 외국인 결제 PoC
~2주
05
법적 자격 — 부동산 중개·전자금융·외환 적법성
CRITICAL 비즈 확신도 75%
시나리오
V1 출시 3개월, 사용자 1명이 호스트 사기 피해로 한국소비자원 신고
소비자원이 우리를 부동산 중개업자로 판단 → 관할 구청 조사 의뢰
공인중개사법 위반(무자격 중개업) 적발 → 영업정지 명령 + 형사 고발
진행 중 정기결제 계약 전건 이행 불능 → 집단 환불 발생 → 유동성 압박
외부 근거
공인중개사법 제2조 — "중개"란 중개대상물(주택·토지·건물 등)에 대한 매매·교환·임대차 그 밖의 권리의 득실변경에 관한 행위를 알선하는 것. 국가법령정보센터 · 공인중개사법
공인중개사법 제48조 — 무자격 중개업: 3년 이하 징역 또는 3천만원 이하 벌금. 동법 벌칙 규정
전자금융거래법 제28조 — 전자지급결제대행(PG)업 등록 의무. 미등록 영업 시 5년 이하 징역 또는 5천만원 이하 벌금. 제3자 자금 수취 후 정산 구조에 적용 가능. 국가법령정보센터 · 전자금융거래법
직방·다방의 중개업 라이선스 논쟁 (2017~2022) — 결국 자회사로 공인중개사법인 보유. "정보 노출만 한다"는 주장이 인정받지 못한 선례. 국토교통부 행정해석 · 부동산 플랫폼 규제 가이드라인
위홈(2021) 호스트 사기 사건 — 단순 중개 플랫폼이라는 항변에도 플랫폼 책임 일부 인정 판례. 서울중앙지법 2021 민사 판결
확신도 75%인 이유
실제 적용은 변호사 판단 필요. "공급자가 면허 보유 + 우리는 기술 서비스" 구조가 어느 정도 인정받을 가능성 있음. 그러나 우리가 결제 수취·UI 통제·계약 동기화 기록까지 하므로 완전 면책은 어려움.
반증 조건
이 발견이 기각되는 조건: 법무법인 의견서가 이미 존재하고 결론이 "기술 서비스로 분류 가능 + 핸디즈 면허 하 운영"이라면 본 항목은 메타 정보(법무 검토 완료)만 PRD에 추가하면 해소.
닫는 법
법무 · 의견서 확보 + PRD에 메타 정보 명시
